# DSA-Analysis-Design

What was the problem that you were trying to solve in the projects for this course?
The problem that I was attempting to solve in the project was to build a simple program that reads a file, parses the data into an object structure, and prints both specific object data, as well as all data stored in objects from the file.

How did you approach the problem?
I approached the problem considering project requirements, memory use, and algorithm runtime. After analyzing data structures such as vectors, binary trees and hash tables, and after analyzing various algorithms that manipulate, sort, and store object data, I came to the conclusion that a hash table paired with a vector was the most efficient structure for the data in the project.

How did you overcome any roadblocks you encountered while going through the activities and projects?
There were many roadblocks while building this project but through careful research, patience, and trial-an-error I was able to overcome them. One of the  most difficult roadblocks was presented early by the developer environment itself. In order to test my program I needed to load a template file that contained a few lines of data. While the file was contained within my project, I found that the file could not be read during debug. After considerable stress and research I learned how to modify the workspace and file directory debug settings and was able to finally test my code.

How has your work on this project expanded your approach to designing programs and developing software?
More than anything, this project has proven to me the importance of planning and design before building anything. The design of systems, data structures, and databases all rely on how fast and efficient an algorithm is to process a given task, and for that reason standard algorithms with varying runtimes have been established to accomplish different tasks in different scenarios, with different systems. Proper planning and design on how an algorithm affects the memory and runtimes of a system improves workflow processes and prevents costly rewrites and system failure.

How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?
One great benefit that has come from this project and its activities is that I have greatly improved my ability to write clean pseudocode. I had previously had a propensity to include symbols and syntax that is present in actual code, like semi-colons and brackets, but have learned that if its presented properly with whitespace and indentation, none of that is necessary and the reader can still determine where a function begins and ends. Another benefit has been my increased ability to see where duplicate code can be made more maintainable and adaptable by building reusable methods. Together with valuable feedback from the professor, this project has noticeable improved my programming abilities.
